A Survey of Efficient Transformer Architectures for Large-Scale NLP

Authors: Zhang, Wei; Patel, Anika; Morrison, James
Published: JMLR Vol. 26, 2025 | DOI: 10.1234/jmlr.2025.efficient-transformers

Abstract

This paper surveys efficient transformer architectures: sparse attention, linear attention approximations, memory-efficient training, and hardware-aware design. We cover 200+ papers from 2020-2025.

1. Introduction

The transformer (Vaswani et al., 2017) dominates NLP but scales quadratically with sequence length. Training GPT-3 required ~3.14e23 FLOPS. This motivates efficient alternatives.

2. Sparse Attention

Sparse patterns restrict attended token pairs. Longformer uses local windows, BigBird adds global+random tokens. Learned sparsity (Routing Transformer, Reformer) adapts dynamically. Combining local+global tokens works best.

3. Linear Attention

Performer uses FAVOR+ for O(n) attention. State space models (S4, Mamba) achieve linear complexity. Hybrids like Jamba interleave Mamba+attention layers.

4. Memory-Efficient Training

Gradient checkpointing trades 20-30% compute for 5x memory savings. FlashAttention achieves O(n) memory with 2-4x speedup via IO-aware tiling. Mixed precision (BF16) halves memory.

5. Conclusion

Efficient transformers combine multiple strategies co-designed with hardware. Hybrid attention+SSM architectures are most promising.
